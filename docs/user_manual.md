## 1. 快速入门

### 1.1 登录与账户

**首次访问**：
1. 在浏览器打开平台地址（联系管理员获取 URL）
2. 点击"注册"，填写用户名、密码、机构名称
3. 注册成功后自动登录，系统为每个机构创建独立的租户空间，数据完全隔离

**登录**：
- 输入用户名和密码登录
- 支持 Token 自动刷新（Access Token 15分钟有效，过期自动刷新）

**订阅计划**：
- 基础版：按实际 AI 调用量扣点（每 1000 tokens 消耗 1 点）
- 年费会员：免扣点，享受专属服务

### 1.2 界面导航

顶部主菜单顺序：
1. **AI 助手**（核心交互入口，位于工作台上方）
2. **工作台**（Dashboard，登录后默认首页）
3. **项目管理**
4. **主体管理**（境内主体 / 境外标的）
5. **AI 报告**
6. **合规规则**
7. **计费中心**（需登录）
8. **LLM 配置**（仅管理员可见）

## 2. 工作台

工作台是登录后的默认首页，展示业务概览和快捷入口。

**统计卡片**：
- 项目总数：当前所有项目数量
- 进行中：处于非完成状态的项目数量
- 剩余点数：账户当前可用 AI 点数
- 已完成：已投后管理的项目数量

**项目状态分布**：以进度条形式展示各状态的项目数量及占比。

**快捷操作**：
- 新建 ODI 项目：跳转项目创建页面
- 管理境内主体：跳转境内主体管理
- AI 报告生成：跳转 AI 报告中心
- 查看计费详情：跳转计费中心

**账户信息**：显示机构名称、订阅计划类型（年费会员/基础版）、剩余点数余额。

## 3. 项目管理

项目管理是 ODI 平台的核心功能，所有投资项目都在此处管理，支持全生命周期跟踪。

### 3.1 创建新项目

入口：菜单 → 项目管理 → 新建项目

**必填信息**：
- 境内主体：从已添加的境内主体中选择（投资方）
- 境外标的：从已添加的境外标的中选择（被投资方）

**可选信息**：
- 项目名称：留空将自动生成，格式为 `[境内主体]在[国家]新设[境外企业]项目`
- 投资金额：数字，单位为所选币种
- 币种：USD（美元）/ CNY（人民币）/ HKD（港元）/ EUR（欧元）
- 投资架构：
  - `DIRECT`：直接投资（境内主体直连境外标的）
  - `SPV_HK`：香港 SPV（经香港特殊目的实体中转）
  - `SPV_SGP`：新加坡 SPV（经新加坡特殊目的实体中转）
  - `MULTI_LAYER`：多层架构（复杂跨境结构）
- 投资必要性说明：描述投资背景和目的，AI 将基于此内容扩写报告，建议详细填写

### 3.2 查看项目详情

入口：项目列表 → 点击任意项目名称

**项目详情包含**：
- 基本信息（名称、金额、架构、创建时间）
- 状态流水线（9阶段进度条，当前阶段高亮显示）
- AI 生成报告（预审报告、可研报告、尽调报告，可折叠展开）
- 状态流转日志（时间线形式记录每次状态变更、操作人、时间）

### 3.3 推进项目状态

在项目详情页右上角，点击"推进至: [下一状态]"按钮。

**状态推进规则**：
- 每次只能推进到下一个状态，不可跳跃
- 推进到 `NDRC_APPROVED`（发改委通过）时，必须上传发改委备案通知书
- 推进到 `MOFCOM_APPROVED`（商务部通过）时，必须上传商务部企业境外投资证书
- 可填写推进备注（选填）

**文件上传**：支持 PDF、JPG、PNG、DOC、DOCX 格式。

### 3.4 项目九阶段状态机

| 状态 Key | 显示名称 | 说明 |
|---------|---------|------|
| `PRE_REVIEW` | 智能预审 | 项目创建后初始状态，AI 进行合规风险预审 |
| `DATA_COLLECTION` | 材料准备 | 收集整理备案所需材料 |
| `NDRC_FILING_PENDING` | 发改委待备案 | 向国家发改委提交备案申请 |
| `NDRC_APPROVED` | 发改委通过 | 获得发改委备案通知书 |
| `MOFCOM_FILING_PENDING` | 商务部待备案 | 向商务部提交备案申请 |
| `MOFCOM_APPROVED` | 商务部通过 | 获得商务部投资证书 |
| `BANK_REG_PENDING` | 银行登记 | 在银行进行外汇登记 |
| `FUNDS_REMITTED` | 资金汇出 | 完成资金出境汇款 |
| `POST_INVESTMENT` | 投后管理 | 进入投后管理阶段 |

**状态颜色标识**：
- 蓝色：初始阶段
- 青色：材料准备
- 紫色：待备案
- 绿色：已通过/已完成
- 橙色：需关注
- 蓝灰色：银行登记
- 灰色：投后管理

## 4. AI 助手

AI 助手是平台的核心交互入口，通过自然语言对话完成 ODI 相关操作，无需记忆具体功能位置。

### 4.1 功能入口

菜单位置：平台首个菜单项，位于工作台上方。

### 4.2 支持的意图类型

AI 助手通过意图识别（Intent Detection）自动理解用户需求并路由到对应功能：

| 意图 | 标签颜色 | 说明 | 示例对话 |
|------|---------|------|---------|
| `create_project` | 绿色 | 创建新 ODI 项目 | "帮我创建一个去越南投资的项目" |
| `query_project` | 蓝色 | 查询已有项目信息 | "查一下所有进行中的项目" |
| `pre_review` | 橙色 | 执行智能预审 | "对这个项目做一下预审" |
| `generate_report` | 紫色 | 生成可研报告或尽调报告 | "生成一份可行性研究报告" |
| `export_ndrc` | 青色 | 导出发改委 XML 文件 | "导出发改委文件" |
| `export_mofcom` | 青色 | 导出商务部 Excel 文件 | "导出商务部表格" |
| `query_entity` | 蓝灰色 | 查询境内/境外主体信息 | "查询一下境内主体" |
| `query_rules` | 紫红色 | 查询合规规则 | "有哪些高风险国家" |
| `knowledge_qa` | 灰色 | 基于 Dify 知识库的 RAG 问答 | "什么是 ODI？" |
| `general_chat` | 灰色 | 闲聊 | "你好" |

### 4.3 上下文项目关联

右侧边栏可选择"上下文项目"（选填）。

选择项目后，AI 将自动关联该项目的完整上下文（投资金额、目的国、行业等），无需在对话中重复说明项目名称。

### 4.4 附件上传

输入框左侧支持附件上传（回形针图标）。

**支持格式**：PDF、图片（jpg/png）、Word 文档（doc/docx）

上传后，附件将随消息一起发送，AI 可读取附件内容（如 PDF 合同、资质证书）进行回复和分析。

### 4.5 对话历史

对话内容实时保存，可在 AI 助手页面持续查看历史记录。

点击右侧"新对话"按钮可清除当前对话，重新开始新话题。

## 5. AI 报告中心

AI 报告中心提供结构化的报告生成功能，适合需要正式输出文档的场景。

入口：菜单 → AI 报告

**报告类型**：智能预审、可研报告、尽调报告

### 5.1 智能预审

**功能**：对项目进行综合合规风险评估。

**输出内容**：
- 交通灯风险评级：
  - 🟢 LOW（低风险）：建议推进
  - 🟡 MEDIUM（中风险）：需关注
  - 🔴 HIGH（高风险）：谨慎评估
- 命中规则列表：显示触发规则的目标值和风险等级
- AI 综合分析报告：Markdown 格式，包含风险点汇总和建议
- 资产预警（如适用）：针对特定情况的风险提示

**入口**：
- AI 报告中心 → 选择项目 → 点击"智能预审"
- 项目详情页 → AI 生成报告区域 → 点击"智能预审"

### 5.2 可行性研究报告

**功能**：基于项目信息自动生成符合国家发改委格式要求的可行性研究报告。

**报告内容**：
- 投资背景与目的
- 目标市场与行业分析
- 投资架构设计
- 财务预测与资金安排
- 风险分析
- 结论与建议

**导出**：支持导出为 PDF 文件（点击报告右上角"导出 PDF"按钮）。

### 5.3 尽职调查报告

**功能**：生成境外投资尽职调查报告，适用于并购或合资场景。

**报告内容**：
- 目标公司基本情况
- 股权结构分析
- 资产负债概况
- 经营状况评估
- 合规情况审查
- 主要风险点
- 投资建议

**导出**：支持导出为 PDF 文件。

## 8. 表单导出

支持多部委电子表单的自动生成和下载，所有导出均以当前项目数据自动填充字段。

### 8.1 发改委 XML 导出

生成符合国家发改委备案要求的 XML 格式文件，结构化包含项目信息、投资主体信息、目标国信息等。

**适用阶段**：NDRC_FILING_PENDING 及之后

**导出方式**：
- 项目详情页 → 右上角"导出"下拉菜单 → 选择"发改委 XML"
- AI 助手对话："帮我导出发改委文件"

### 8.2 商务部 Excel 导出

生成符合商务部备案要求的 Excel 格式文件，包含自动映射的备案字段。

**适用阶段**：MOFCOM_FILING_PENDING 及之后

**导出方式**：
- 项目详情页 → 右上角"导出"下拉菜单 → 选择"商务部 Excel"

### 8.3 打包下载

一次性导出发改委 XML + 商务部 Excel + 相关附件，打包为 ZIP 文件方便一次性提交。

**导出方式**：项目详情页 → 右上角"导出"下拉菜单 → 选择"全部打包"

## 9. 合规规则管理

配置国家和行业的合规风险规则，智能预审时系统自动匹配并标注风险。

入口：菜单 → 合规规则（仅管理员可编辑，普通用户仅可查看）

**规则类型**：
- `COUNTRY`：国家规则，按目的国区分风险（例：投资到某国的风险等级）
- `INDUSTRY`：行业规则，按投资行业区分风险（例：某行业的投资风险等级）

**风险等级**：
- `HIGH`（高风险）：红色标签，存在明确限制或高不确定性
- `MEDIUM`（中风险）：橙色标签，需额外审批或关注
- `LOW`（低风险）：绿色标签，符合常规审批

**新建规则**（管理员）：
1. 点击"新增规则"
2. 选择规则类型（国家/行业）
3. 填写目标值（如"伊朗"、"房地产"）
4. 选择风险等级
5. 填写规则名称和描述（选填）

**示例规则**：
- 国家规则：目的国 = 伊朗，风险等级 = HIGH
- 国家规则：目的国 = 新加坡，风险等级 = LOW
- 行业规则：行业 = 武器制造，风险等级 = HIGH

## 10. LLM 模型配置

管理 AI 大模型的 API 提供商配置，支持多模型自动路由和故障转移。

入口：菜单 → LLM 配置（仅管理员可见）

**支持提供商**：
- DeepSeek（默认，最高优先级）
- Kimi / Moonshot（备用）
- MiniMax（备用）

**配置字段**：
- 提供商名称：选择 deepseek / kimi / minimax
- 显示名称：自定义显示名称
- API Key：提供商提供的密钥
- Base URL：API 端点地址
- 模型版本：如 deepseek-chat、moonshot-v1-8k 等
- 优先级：数字越小优先级越高（0 为最高）
- 启用状态：开关控制是否启用

**工作原理**：
- 系统按优先级依次尝试可用模型
- 当前一个模型调用失败时，自动降级到下一个（最多 3 次重试，指数退避）
- 所有模型均不可用时返回错误

## 12. 角色与权限

| 角色 | 说明 | 可用功能 |
|------|------|---------|
| ADMIN（管理员） | 平台最高权限 | 全功能访问，包括 LLM 配置、导出模板管理、合规规则编辑、用户管理 |
| OPERATOR（操作员） | 普通业务人员 | 项目管理、AI 报告生成、表单导出、主体管理 |
| VIEWER（查看者） | 只读用户 | 查看项目、报告、数据（不能创建或修改） |

**注意**：
- 普通用户（OPERATOR/VIEWER）无法访问 LLM 配置页面
- 删除操作（项目、主体等）仅管理员可见按钮
- 规则管理仅管理员可编辑，OPERATOR/VIEWER 仅可查看

## 13. 常见问题

**Q1: 创建项目时提示"点数不足"**
A：AI 预审和报告生成会消耗点数。请先到计费中心充值点数，或确认账户已升级为年费会员。

**Q2: 状态推进失败，提示需要上传文件**
A：检查是否满足了推进条件。推进到"发改委通过"需上传备案通知书，推进到"商务部通过"需上传投资证书。上传 PDF/JPG 等格式后再次尝试。

**Q3: AI 报告生成为空或内容不完整**
A：确认项目已填写必要的投资信息（投资金额、目的国、境内/境外主体等）。信息越完整，报告质量越高。可在项目详情页编辑补充信息。

**Q4: 表单导出内容与实际不符**
A：在管理后台（LLM 配置页）检查导出字段映射配置是否与当前模板匹配。字段映射可在管理后台实时调整，无需重启服务。

**Q5: 忘记了密码**
A：请联系平台管理员重置密码。

**Q6: AI 助手回复说"无法理解"**
A：请尽量使用明确的操作描述。有效的意图包括：创建项目、查询项目、智能预审、生成报告、导出发改委/商务部文件、查询主体、查询规则、知识库问答等。例如："创建一个去越南的项目"比"帮我弄一个"更有效。

**Q7: DeepSeek / Kimi / MiniMax 模型调用全部失败**
A：检查管理后台 LLM 配置中的 API Key 和 Base URL 是否正确，确认网络连接正常。系统具备自动降级能力，如所有模型均不可用请联系平台管理员检查配置。
